1.1 Modelos de Arquitecturas de Cómputo
1.1.1 Clásicas

Arquitectura Von Neumann

Modelo que integra CPU, memoria y dispositivos de E/S en un sistema centralizado con un flujo secuencial de instrucciones.

Arquitectura Harvard

Modelo con memorias separadas para datos e instrucciones, permitiendo acceso simultáneo.

Los modelos clásicos de arquitectura de cómputo

Son las estructuras básicas que dieron origen a las computadoras modernas. Se centran en cómo se conectan y comunican los componentes principales como el procesador, la memoria y los dispositivos de entrada/salida.

Arquitectura de Von Neumann:

  1. Usa una sola memoria para datos e instrucciones.
  2. Ejecuta las instrucciones secuencialmente.
  3. Es el modelo más común en computadoras personales.
  4. Su principal limitación es el cuello de botella de Von Neumann, debido a que solo puede acceder a datos o instrucciones en un solo ciclo.

Arquitectura Harvard:

  1. Separa la memoria de datos y de instrucciones.
  2. Permite mayor velocidad al poder acceder a ambas simultáneamente.
  3. Común en sistemas embebidos y microcontroladores.
1.1.2 Segmentadas

Sistemas que dividen el procesamiento en etapas paralelizables (pipelines) para aumentar el rendimiento.

  • Pipeline de instrucciones
  • Pipeline aritmético
  • Pipeline gráfico

Arquitecturas Segmentadas

También conocidos como arquitecturas segmentadas o pipeline, estos modelos mejoran el rendimiento al dividir la ejecución de instrucciones en etapas que se pueden realizar en paralelo.

Funcionan como una línea de ensamblaje: mientras una instrucción se está decodificando, otra puede estarse ejecutando.

Las etapas comunes del pipeline incluyen:

  • Fetch (búsqueda)
  • Decode (decodificación)
  • Execute (ejecución)
  • Memory access
  • Write-back

Aumentan la velocidad sin necesidad de duplicar recursos, aunque pueden presentar problemas de interrupciones o dependencias entre instrucciones.

1.1.3 De multiprocesamiento

Arquitecturas con múltiples procesadores que trabajan cooperativamente:

  • SMP (Symmetric Multi-Processing)
  • NUMA (Non-Uniform Memory Access)
  • Cluster computing

Modelos de Multiprocesamiento

Estos modelos utilizan más de un procesador para ejecutar múltiples tareas al mismo tiempo, mejorando significativamente el rendimiento.

SMP (Symmetric Multiprocessing):

Todos los procesadores comparten la misma memoria y tienen acceso equitativo a los recursos.

MPP (Massively Parallel Processing):

Cada procesador tiene su propia memoria y se comunican mediante una red.

Son ideales para tareas que requieren alto rendimiento como:

  • Simulaciones
  • Bases de datos grandes
  • Aplicaciones científicas

Aprovechan el paralelismo a nivel de proceso o hilo para ejecutar múltiples instrucciones simultáneamente.

1.2.2.1 Conceptos Básicos del manejo de memoria

  • Direccionamiento

Memoria

Es un componente esencial para el almacenamiento y acceso a datos e instrucciones.

Memoria principal (RAM):

De acceso rápido y volátil; almacena temporalmente datos e instrucciones en uso.

Memoria caché:

Pequeña y rápida; acelera el acceso a datos frecuentes.

Memoria ROM:

No volátil; contiene instrucciones básicas como el arranque del sistema.

Manejo de Entrada/Salida (E/S)

Permite la comunicación entre la computadora y el entorno externo (usuario o dispositivos).

Puede ser:

  • Programado
  • Por interrupciones
  • Mediante acceso directo a memoria (DMA)

Involucra dispositivos como:

  • Teclado
  • Mouse
  • Disco duro
  • Impresora
  • Etc.

La eficiencia del sistema depende en parte de cómo se gestionan estas operaciones.

1.2.4 Buses
1.2.4.1 Tipos de buses

Clasificación de buses:

  • Bus del sistema
  • Bus de expansión
  • Bus local
  • Bus de backplane

Buses

Son canales que transportan datos, direcciones e instrucciones entre los componentes del sistema.

Bus de datos:

Transfiere la información procesada.

Bus de direcciones:

Indica la ubicación de la memoria o dispositivo al que se accede.

Bus de control:

Coordina las operaciones del sistema.

Su ancho (número de líneas) afecta la cantidad de datos que pueden transmitirse simultáneamente.

1.2.5 Interrupciones

Mecanismo para alterar la secuencia normal de ejecución:

  • Interrupciones hardware vs software

  • Manejo de excepciones

Interrupciones

Son señales que alteran temporalmente el flujo de ejecución del CPU para atender eventos urgentes.

Pueden ser:

  • Hardware (como una tecla presionada)
  • Software (errores o solicitudes del sistema)

El procesador:

  1. Detiene la tarea actual
  2. Atiende la interrupción
  3. Luego la reanuda

Permiten una respuesta rápida y eficiente a eventos externos sin afectar el rendimiento general del sistema.
