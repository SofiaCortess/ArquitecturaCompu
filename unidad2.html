Unidad 2: Estructura y funcionamiento de la unidad central de procesamiento

2.1 Organización del procesador
2.1.1 Registros visibles para el usuario

Registros accesibles por el programador en lenguaje ensamblador:

  • Registros de propósito general: AX, BX, CX, DX en x86
  • Registros índice: SI, DI (Source Index, Destination Index)
  • Registros de segmento: CS, DS, SS, ES
  • Registro puntero de instrucción (IP): Indica la siguiente instrucción a ejecutar
  • Registro de banderas (FLAGS): Almacena el estado del procesador
2.2 Registros de control y de estados
2.2.1 Ejemplos de registros de CPU reales

Registros internos que controlan el funcionamiento del procesador:

  • Registro de estado (PSW): Contiene bits de condición (zero, carry, overflow)
  • Registro de control: Habilitación de interrupciones, modo kernel/user
  • MAR (Memory Address Register): Dirección de memoria a acceder
  • MBR (Memory Buffer Register): Datos leídos/escribir en memoria
  • Registro de instrucción (IR): Instrucción actual siendo ejecutada

Ejemplos en arquitecturas reales:

  • x86: CR0-CR4 (Control Registers)
  • ARM: CPSR (Current Program Status Register)
  • MIPS: CP0 (Coprocessor 0 registers)
2.3 El ciclo de instrucción
2.3.1 Ciclo fetch-decode-execute

Fases básicas de ejecución de una instrucción:

  1. Fetch: Obtener la instrucción de memoria
  2. Decode: Decodificar la instrucción
  3. Execute: Ejecutar la operación
  4. Store: Almacenar resultados (en algunos casos)

Duración típica: 1-4 ciclos de reloj por instrucción (CPI)

2.3.2 Segmentación de instrucciones

Técnica para ejecutar múltiples instrucciones simultáneamente en diferentes etapas:

  • Pipeline clásico de 5 etapas: IF, ID, EX, MEM, WB
  • Riesgos:
    • Estructurales (recursos ocupados)
    • De datos (dependencias entre instrucciones)
    • De control (branches/jumps)
  • Técnicas de optimización: Forwarding, branch prediction
2.3.3 Conjunto de instrucciones: Características y funciones

Clasificación de instrucciones del CPU:

  • Por formato: RISC vs CISC
  • Por función:
    • Aritmético-lógicas (ADD, SUB, AND, OR)
    • Transferencia de datos (MOV, LOAD, STORE)
    • Control de flujo (JMP, CALL, RET)
    • Entrada/salida (IN, OUT)
  • Modos de operación: Supervisor vs usuario
2.3.4 Modos de direccionamiento

Formas de especificar operandos en las instrucciones:

  • Registro: Operando en registro (ADD R1, R2)
  • Inmediato: Valor constante (ADD R1, #5)
  • Directo: Dirección de memoria absoluta (LOAD R1, 0x1000)
  • Indirecto: Dirección en registro (LOAD R1, (R2))
  • Indexado: Dirección base + desplazamiento (LOAD R1, 100(R2))
  • Relativo al PC: Para saltos (JMP +0x50)
2.4 Casos de estudio de CPU reales

Análisis comparativo de arquitecturas reales:

Intel Core i9 (CISC)

  • Arquitectura x86-64
  • Pipeline de 14-19 etapas
  • Ejecución fuera de orden
  • HyperThreading

ARM Cortex-A78 (RISC)

  • Conjunto reducido de instrucciones
  • Pipeline de 11 etapas
  • Diseño power-efficient
  • Big.LITTLE architecture

AMD Ryzen 9

  • Arquitectura Zen 3
  • Chiplet design
  • Unified L3 cache
  • Precision Boost 2

Apple M1

  • ARM-based SoC
  • Unified memory architecture
  • 8-core CPU (4 performance + 4 efficiency)
  • Neural Engine
